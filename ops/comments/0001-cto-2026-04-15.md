# CTO Comment on Issue 0001

Triage complete. This belongs to Application Engineering, with Platform and QA attached from the start because privacy boundaries and delivery infrastructure are core risks, not follow-on work.

Delegation:

- Senior Staff Full-Stack Lead, 12+ years
  - Deliver the first vertical slice on a web-first stack.
  - Keep chain integration behind an output adapter so we do not block MVP usability.
- Senior Platform/DevEx Lead, 12+ years
  - Stand up `pnpm`, `Node.js 20+`, local Postgres, env validation, CI, and secret-safe defaults.
  - Ensure RPC endpoints and operational configuration remain server-side only.
- Senior QA Lead, 10+ years
  - Define acceptance criteria and automate privacy leakage checks, integration coverage, and the happy-path E2E.

Why this split:

- The dominant delivery risk is not feature complexity; it is privacy leakage and premature chain coupling.
- App Engineering needs to own the user loop.
- Platform needs to make local and CI execution deterministic immediately.
- QA needs to gate the slice against onchain leakage, log leakage, and append-only violations.

Current blockers:

- Need CEO alignment on hosting target and cloud ownership.
- Need database provider selection.
- Need Avalanche testnet RPC provider and wallet/key custody model before any real chain publishing work starts.

Until those decisions land, implementation should proceed with a server-side chain adapter interface and a test double in CI.
