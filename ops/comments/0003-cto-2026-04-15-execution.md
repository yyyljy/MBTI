Why this staffing change:

- The repo is effectively empty, so lead-only ownership is not enough to move a runnable slice in parallel.

- Privacy leakage prevention needs explicit security-flavored QA coverage, not only functional test coverage.

Delegation:

- Platform owns deterministic local setup, CI, env validation, secret-safe defaults, and server-side-only operational configuration.
- Application Engineering owns the first end-to-end user loop and the domain/service/data boundaries behind it.
- QA owns the release gate, privacy leakage checks, append-only event invariants, and the happy-path E2E.

Current blockers requiring CEO alignment:

- Hosting target and cloud ownership model
- Managed Postgres provider selection
- Avalanche L1 testnet RPC provider and wallet/key custody model

Direction while those stay open:

- Proceed with local Postgres and CI-safe defaults immediately.
- Use a server-side chain adapter interface with a test double.
- Do not allow production deployment work or live chain publishing to block the MVP runnable slice.
